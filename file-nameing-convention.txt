‚úÖ Components

Use PascalCase (a.k.a. UpperCamelCase) for React components:

HomeScreen.tsx
UserProfileCard.tsx
LoginButton.tsx
AppNavigator.tsx


Each component gets its own folder if it has multiple related files:

components/
  UserProfileCard/
    index.tsx
    styles.ts
    types.ts


Or:

components/
  UserProfileCard.tsx

‚úÖ Hooks

Use camelCase and always start with use:

useFetchData.ts
useTheme.ts

‚úÖ Utility / Helper Files

Use camelCase:

formatDate.ts
calculateTotal.ts
validateEmail.ts

‚úÖ Stylesheets

Use camelCase or styles.ts when local to a component:

HomeScreen.styles.ts
styles.ts

‚úÖ Context / Providers

Use PascalCase for context providers:

AuthContext.tsx
ThemeProvider.tsx

‚úÖ Navigation Files

Use PascalCase for navigators:

AppNavigator.tsx
AuthStackNavigator.tsx
MainTabNavigator.tsx

‚úÖ Types / Interfaces

Use PascalCase for TypeScript files that define types:

types.ts
UserTypes.ts
navigationTypes.ts

‚úÖ Config / Constants

Use UPPER_CASE for exported constants, and file names in camelCase:

apiConfig.ts
appConstants.ts
colors.ts

‚úÖ Assets

For images, fonts, etc., use kebab-case:

  user-avatar.png
  app-logo.png
